F1 Race Calendar Optimization


On this page, you will be able to compare the FOL TSP model to a few heuristic models that also find the shortest path among selected cities. To show the comparison better, this page utilizes the capitals of the continental US states rather than the 2025 F1 venues.


Nearest Neighbor: This model finds the shortest path by starting at a specific city then visiting the city closest to it. This process repeats until all the cities are visited. If a specific start city is not chosen, then the model will test the algorithm with each of the selected cities as the start city then return the shortest path found. Due to the nature of this model, a specific end city cannot be chosen.


Nearest Insertion: This model finds the shortest path by starting at a specific city then visiting the city closest to it. However, the next iteration compares the closest cities to both cities on the end of the current path and visits the closest city of the two. This process repeats until all the cities are visited. Though you will have the option to select a start city, that city may not necessarily be the first city in the path as the mdoel looks at both ends of the developing path. And similar to the Nearest Neighbor model, if a specific start city is not chosen, then the model will test the algorithm with each of the selected cities as the start city then return the shortest path found. Due to the nature of this model, a specific end city cannot be chosen.


Deleted TSP: This model finds the shortest path by finding the shortest route (loop) that visits every city using the TSP model then removing the longest edge. Due to the nature of this model, a specific start city and end city cannot be chosen.


If all the options are left blank, then the FOL TSP model will find the shortest path among all the cities..
